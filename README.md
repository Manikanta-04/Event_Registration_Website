## 🧠 Problem Statement

Event registration websites are typically generic, lifeless forms that fail to capture the excitement of the event they represent. Current problems:

- 😐 Bland registration pages with no thematic connection to the event
- 🎨 No immersive visual experience — just forms and tables
- 📵 Heavy frameworks required for even simple animated UIs
- 🦸 Fan-event platforms miss the opportunity to make attendees *feel* like they're part of the universe

**An event as exciting as the Ben 10 Universe Summit deserves a registration experience that feels like wielding the Omnitrix.**

---

## 💡 Solution

**Ben 10 Universe Summit** is a fully self-contained, single HTML file event registration platform — featuring an animated 3D Omnitrix, alien identity selector, mission schedule timeline, animated counters, and a fully functional registration form — built entirely with pure HTML, CSS & JavaScript. No frameworks. No build step.

## 📊 System Design

```
Page Load Flow:

[Browser opens index.html]
         │
         ▼
[CSS loads → Omnitrix entrance animation begins]
[JS executes → particles generated, stars placed]
         │
         ▼
[User scrolls → stats counters animate on viewport entry]
[Alien cards → mousemove triggers 3D tilt via JS]
         │
         ▼
[User selects alien → card highlights + selection stored]
[User fills registration form → validation runs on submit]
         │
         ▼
[Valid submit → success overlay animates in]
[Invalid submit → field-level error messages shown]
```

```
JavaScript Modules:

counterAnimation()    → requestAnimationFrame count-up from 0 to target
particleGenerator()   → creates 80+ floating DOM particles, loops via CSS
starField()           → places 200 twinkling star elements in background
alienTilt()           → mousemove → calculate angle → apply rotateX/Y
formHandler()         → validate all fields → show success overlay
```

## 🔄 Workflow

```
1. User opens index.html / visits live URL
2. Omnitrix 3D entrance animation plays (~2s)
3. Floating particles + star field initialize
4. User scrolls down → stat counters animate up
5. User selects alien identity → card highlights
6. User views mission schedule timeline
7. User fills registration form (name, email, pass, alien)
8. Form validated → success overlay displays
9. User can register for the Universe Summit ✅
```
